# Verification: `.h5` + `.mem` for student chip design

## Verdict (short)

| Question | Answer |
|----------|--------|
| **Need to retrain?** | **No** ‚Äî float accuracy is already locked in `model_full.pt`. |
| **Can we produce `.h5` + `.mem` now?** | **Yes** ‚Äî by **export/convert** from `.pt` (tool: `tools/export_h5_mem_for_fpga.py`). |
| **Are those two files ‚Äúfull standard-ready‚Äù for every RTL team?** | **Partially.** Format is industry-usable; **bit-width, address map, and quant scheme must still match the student RTL**. That is a **hardware contract**, not a training gap. |

---

## What `.h5` and `.mem` mean in this context

### 1. `.h5` (HDF5)

| Meaning in literature / courses | Our export |
|---------------------------------|------------|
| Keras `model.save("x.h5")` ‚Äî **full graph + weights** | **Not the same** without a matching Keras architecture |
| Weight bank / scientific HDF5 of tensors | **Yes** ‚Äî `model_weights.h5` with groups per tensor, float32 |
| Input to hls4ml / some HLS flows | Needs **Keras/ONNX graph**, not only a weight dump |

**Standard we meet:** valid HDF5 file, layer-named datasets, float32 weights+BN+classifier, gzip, documented in manifest.

**Standard we do not claim:** `tf.keras.models.load_model("model_weights.h5")` works out of the box (no Keras graph stored).

### 2. `.mem` (FPGA memory init)

| Meaning | Our export |
|---------|------------|
| Vivado **`$readmemh`** hex memory file | **Yes** ‚Äî one 16-bit hex word per line |
| BRAM/ROM initialization of weights | **Yes** ‚Äî concatenated regions with address map in JSON |
| Single ‚Äúmagical‚Äù format all chips use | **No universal standard** ‚Äî word width (8/16/32), Q-format, row-major vs neuron-major, endianness depend on **your** RTL |

**Standard we meet:** classic Xilinx/Vivado hex `.mem` for simulation/synthesis init ([Vivado $readmemh usage is common for BRAM init](https://adaptivesupport.amd.com/)).

**Must agree with chip team:**

- Word width (we default **INT16**)
- Per-tensor scale (stored in `export_h5_mem_manifest.json`)
- Order of tensors (sorted parameter names)
- Whether BN is fused offline into conv scales (we export **raw** BN Œ≥/Œ≤ separately)

## Is current training ‚Äúenough‚Äù?

```text
Training (done)  ‚Üí  model_full.pt (float)  ‚Üí  convert  ‚Üí  model_weights.h5
                                                   ‚îî‚Üí  model_weights.mem (INT16)
```

| Condition | Status |
|-----------|--------|
| Topology frozen (DS-Conv2D-H1 Pyramid) | Yes |
| Float checkpoint accuracy known (79.08% / 80.00%) | Yes |
| Weights loadable in PyTorch | Yes |
| `.h5` + `.mem` on disk for students | Generated by export tool (see `deploy/student_models/`) |
| Bit-exact match to a specific student RTL | **Only after** they fix address map + quant + compare vectors |
| Keras-identical `.h5` for hls4ml | **Not automatic** ‚Äî optional future path (rebuild in Keras/ONNX) |

**Retrain is NOT required** to create the two files.  
**Retrain (QAT) is only required** if INT16/INT8 on hardware loses too much accuracy versus float.

## Compliance checklist

| Requirement | Met? | Notes |
|-------------|------|-------|
| One `.h5` per model | Yes | Weight bank HDF5 |
| One `.mem` per model | Yes | `$readmemh` hex INT16 |
| Includes weights | Yes | All conv / linear kernels |
| Includes bias-like terms | Yes | BN Œ≤ + FC bias (and BN Œ≥ as scale) |
| Documented layer order / addresses | Yes | manifest `mem.regions` |
| No retrain for export | Yes | From `model_full.pt` |
| Drop-in for arbitrary Verilog NN core | **Conditional** | Core must use our layout or remap |
| Official ‚ÄúIEEE single format for all FPGA courses‚Äù | **No such global standard** | Project must fix RTL interface |

## Recommendation for the chip team

1. Treat **`model_full.pt`** as accuracy reference (software golden).  
2. Use **`.h5`** for analysis / MATLAB / custom quant scripts.  
3. Use **`.mem` + manifest** to fill BRAM in Vivado sim.  
4. Run **bit-true or near-bit-true test vectors** (few clips) before claiming DPU/FPGA accuracy.  
5. Only if step 4 fails badly ‚Üí QAT retrain, then re-export `.h5`/`.mem`.

References (format context): FPGA BRAM `.mem` init practices; Keras/HDF5 weight storage; academic pipelines that train ‚Üí `.h5` ‚Üí extract ‚Üí `.mem` for ROM (e.g. BNN FPGA export literature).
